--------------------------------------------------------------------------------
1. ОБЩИЕ ПОЛОЖЕНИЯ
--------------------------------------------------------------------------------
Все системные события, ошибки и действия пользователей записываются в логи.
Логи используются для:
  - Отладки и поиска ошибок (debugging)
  - Аудита безопасности (security audit)
  - Анализа производительности (performance monitoring)
  - Восстановления истории действий (history tracking)

--------------------------------------------------------------------------------
2. УРОВНИ ЛОГИРОВАНИЯ (LOG LEVELS)
--------------------------------------------------------------------------------
[DEBUG]   - Подробная информация для разработчиков (переменные, запросы)
[INFO]    - Стандартные события (вход пользователя, загрузка страницы)
[WARNING] - Предупреждения (некритичные ошибки, устаревшие данные)
[ERROR]   - Критические ошибки (сбой БД, недоступность сервиса)
[CRITICAL]- Системные сбои, требующие немедленного вмешательства

--------------------------------------------------------------------------------
3. ФОРМАТ ЗАПИСИ
--------------------------------------------------------------------------------
Стандартная строка лога выглядит следующим образом:

  [TIMESTAMP] [LEVEL] [MODULE] - MESSAGE

Пример:
  [2024-01-15 14:30:22] [INFO] [AUTH] - User login successful: user_id=123
  [2024-01-15 14:31:05] [ERROR] [DB] - Connection timeout after 30s

--------------------------------------------------------------------------------
4. ХРАНЕНИЕ И РОТАЦИЯ
--------------------------------------------------------------------------------
- Путь к логам: /var/log/myproject/
- Имя файла: app_YYYY-MM-DD.log
- Ротация: Ежедневно в 00:00
- Хранение: 30 дней (после чего архивируются или удаляются)
- Макс. размер файла: 100 MB (при превышении — принудительная ротация)

--------------------------------------------------------------------------------
5. БЕЗОПАСНОСТЬ И КОНФИДЕНЦИАЛЬНОСТЬ
--------------------------------------------------------------------------------
ЗАПРЕЩЕНО записывать в логи:
  ! Пароли в открытом виде
  ! Полные номера кредитных карт
  ! Персональные данные (ПДН) без маскирования
  ! Токены доступа и секретные ключи

ПРИМЕР МАСКИРОВАНИЯ:
  - Пароль: "password": "******"
  - Карта: "card": "**** **** **** 4242"
  - Email: "email": "u***@example.com"

--------------------------------------------------------------------------------
6. ДОСТУП К ЛОГАМ
--------------------------------------------------------------------------------
- Разработчики: Чтение (только уровень DEBUG/INFO в среде разработки)
- DevOps: Чтение/Запись (все уровни на сервере)
- Администратор: Чтение (уровни WARNING/ERROR/CRITICAL)
- Пользователи: Нет доступа

--------------------------------------------------------------------------------
7. ПРИМЕРЫ КОДА (LOGGING)
--------------------------------------------------------------------------------
// JavaScript пример
console.log(`[INFO] [AUTH] - User ${userId} logged in`);

// Python пример
logger.error(f"[DB] - Connection failed: {error_message}")

// PHP пример
error_log("[WARNING] [API] - Rate limit exceeded for IP: " . $ip);
